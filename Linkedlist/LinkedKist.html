Linked List in Java

A Comprehensive Guide with Detailed Explanations and Visual Examples

What is a Linked List?

A linked list is a linear data structure where elements are not stored in contiguous memory locations. Instead, each element (called a node) contains:

  • A data field that stores the value
  • A pointer/reference to the next node in the sequence

Key Characteristics

Dynamic Size: Unlike arrays, linked lists can grow and shrink during runtime.

Memory Efficiency: Memory is allocated as needed, reducing wasted space.

Insertion/Deletion Efficiency: Adding or removing elements at the beginning is O(1) time.

Types of Linked Lists

Singly Linked List

Each node contains data and a pointer to the next node. Traversal is only in one direction.

class Node {
  int data;
  Node next;
}

Doubly Linked List

Each node contains data, a pointer to the next node, and a pointer to the previous node.

class Node {
  int data;
  Node next;
  Node prev;
}

Circular Linked List

The last node points back to the first node, forming a circle. Can be singly or doubly linked.

// Last node points to head
tail.next = head;

Implementing a Singly Linked List in Java

Here's a complete implementation of a singly linked list in Java:

// Node class definition
class Node {
  int data;
  Node next;

  // Constructor to create a new node
  Node(int d) {
    data = d;
    next = null;
  }
}

// Linked list class
class LinkedList {
  Node head; // head of list

  // Insert new node at the beginning
  public void insertAtBeginning(int data) {
    Node newNode = new Node(data);
    newNode.next = head;
    head = newNode;
  }

  // Insert new node at the end
  public void insertAtEnd(int data) {
    Node newNode = new Node(data);
    if (head == null) {
      head = newNode;
      return;
    }
    Node last = head;
    while (last.next != null) {
      last = last.next;
    }
    last.next = newNode;
  }

  // Delete a node by key
  public void deleteNode(int key) {
    Node temp = head, prev = null;

    // If head node itself holds the key
    if (temp != null && temp.data == key) {
      head = temp.next; // Changed head
      return;
    }

    // Search for the key to be deleted
    while (temp != null && temp.data != key) {
      prev = temp;
      temp = temp.next;
    }

    // If key was not present in list
    if (temp == null) return;

    // Unlink the node from linked list
    prev.next = temp.next;
  }

  // Print the linked list
  public void printList() {
    Node temp = head;
    while (temp != null) {
      System.out.print(temp.data + " -> ");
      temp = temp.next;
    }
    System.out.println("null");
  }
}

Operations and Time Complexity

Operation Description Time Complexity
Insertion at beginning Add a node at the start of the list O(1)
Insertion at end Add a node at the end of the list O(n) - without tail pointer
Insertion at position Add a node at a specific position O(n)
Deletion at beginning Remove the first node O(1)
Deletion at end Remove the last node O(n)
Deletion by value Remove a node with specific value O(n)
Search Find a node with specific value O(n)
Access by index Get the node at a specific position O(n)

When to Use Linked Lists

Advantages

Dynamic Size: No need to pre-allocate memory

Efficient Insertions/Deletions: Especially at the beginning

Memory Usage: Allocates memory as needed

Implementation Flexibility: Foundation for stacks, queues, etc.

Disadvantages

Random Access: No direct access to elements (must traverse)

Memory Overhead: Extra memory for pointers

Cache Performance: Poor locality of reference

Reverse Traversal: Difficult in singly linked lists

Linked Lists vs Arrays

Use linked lists when:

  • You need constant-time insertions/deletions at the beginning
  • The size is unknown and changes frequently
  • You don't need random access to elements

Use arrays when:

  • You need random access to elements
  • You know the number of elements in advance
  • Memory is a critical concern

Key Concepts

Head Pointer

The reference to the first node in the list. Crucial for accessing the entire list.

Node Structure

Each node contains data and a pointer to the next node. The last node points to null.

Traversal

Process of visiting each node starting from the head until reaching null.

Memory Allocation

Nodes are dynamically allocated in memory, which can be non-contiguous.

Common Use Cases

Implementing Stacks

LIFO (Last-In-First-Out) structure with push/pop operations at head.

Implementing Queues

FIFO (First-In-First-Out) structure with enqueue at tail and dequeue at head.

Undo Functionality

Maintaining a history of operations where the most recent is at the head.

Polynomial Representation

Each node represents a term in a polynomial equation.

Java Collections

Java provides a built-in LinkedList class in java.util package:

// Import the LinkedList class
import java.util.LinkedList;

// Create a LinkedList
LinkedList<String> list = new LinkedList<>();

// Add elements
list.add("A");
list.addFirst("B");
list.addLast("C");
